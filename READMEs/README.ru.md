## Проблема

Вы установили множество навыков. Теперь каждый раз, вводя `/`, приходится пролистывать весь список, прежде чем воспользоваться хоть одним из них.

## Решение

skill-habit отслеживает каждый вызванный навык (только метаданные — содержимое промптов никогда не записывается).

При каждом старте сессии система перестраивает префикс ярлыков `/sh-*`, помещая ваши топ-навыки в начало с подписями на любом удобном языке.

Ваш самый используемый навык становится `/sh-<ваш-навык>`, а тот, которым вы не пользовались неделями, опускается в конец.

Список отражает ваши реальные привычки, а не бессмысленный порядок по умолчанию.

## 🔒 Приватность

skill-habit **никогда не записывает** содержимое промптов, пути к файлам или имена проектов.

Записи в логе выглядят так (`~/.skill-habit/skill-usage.log`):

```jsonc
{
  "v": 1,                    // версия схемы
  "ts": 1719360000,          // Unix-метка времени
  "skill": "git-smart",      // только имя — не то, что вы ввели после
  "platform": "claude_code", // какой AI-инструмент вызвал навык
  "hour": 14,                // час суток (0–23)
  "weekday": 1,              // 0 = Понедельник
  "session_id": "abc123",    // идентифицирует непрерывную сессию
  "session_skill_seq": 3,    // позиция в сессии (1, 2, 3…)
  "project_hash": "a3f2c1",  // односторонний хэш, необратим
  "args_len": 42             // только количество символов, никогда не содержимое
}
```

Записи старше `log_retention_days` (по умолчанию: 30) автоматически удаляются при каждой сессии. Также можно вручную очистить данные, введя число дней (0 = удалить всё) в разделе **Настройки → Управление данными**.

## 🚀 Быстрый старт

> **Требования:** macOS или Linux · Python 3.9+ · git
>
> **Windows:** Нативная поддержка пока отсутствует. В качестве обходного пути используйте [WSL](https://learn.microsoft.com/windows/wsl/).

### Установка

**Вариант A — установка через плагин Claude Code (рекомендуется)**

```bash
claude plugins install github:kiss4u/skill-habit
```

Перезапустите Claude Code после установки — отслеживание начнётся немедленно.

**Вариант B — установка одной командой**

```bash
curl -sSL https://raw.githubusercontent.com/kiss4u/skill-habit/main/scripts/bootstrap.sh | bash
```

**Вариант C — pipx / pip**

> Примечание: этот способ не устанавливает встроенные навыки (`/skill-habit:quick`, `/skill-habit:server` и др.). Рекомендуется вариант A или B.

pipx (рекомендуется — изолированная установка):
```bash
pipx install git+https://github.com/kiss4u/skill-habit
skill-habit install
```

pip:
```bash
pip install git+https://github.com/kiss4u/skill-habit
skill-habit install
```

> Запустите новую сессию Claude Code — отслеживание начнётся немедленно.

**Другие платформы**

Поддержка Cursor, Codex CLI, Gemini CLI и других — в планах. Как добавить адаптер — см. раздел «Участие в разработке».

**Открытие панели управления**

После установки откройте панель любым из способов:

В Claude Code:
```bash
/skill-habit:server
```

Установка через pip / pipx:
```bash
skill-habit server
```

Установка через bootstrap / клонирование репозитория:
```bash
python3 ~/.local/share/skill-habit/ui/server.py
```

Браузер откроется автоматически. Сервер завершается через 5 минут простоя.

### Обновление

Используйте ту же команду, что и при установке — каждый способ обрабатывает обновления автоматически:

**Вариант A — плагин Claude Code**

```bash
claude plugins update skill-habit
```

**Вариант B — одна команда**

```bash
curl -sSL https://raw.githubusercontent.com/kiss4u/skill-habit/main/scripts/bootstrap.sh | bash
```

Повторный запуск bootstrap на уже установленном экземпляре обнаружит директорию `.git` и выполнит `git pull` вместо повторного клонирования.

**Вариант C — pipx / pip**

pipx:
```bash
pipx upgrade skill-habit
skill-habit install
```

pip:
```bash
pip install --upgrade git+https://github.com/kiss4u/skill-habit
skill-habit install
```

Также можно проверить наличие обновлений и обновиться в один клик прямо из панели **Настройки → Версия и обновление**.

### Настройка

**Префикс ярлыка**

**Префикс ярлыка** — это пространство имён для всех ярлыков, генерируемых skill-habit; по умолчанию используется `sh`. При каждой сессии ярлыки формируются в одном или обоих режимах (настраивается):

| Режим         | Формат                    | Пример (префикс `sh`)     | Примечание                                                                                                                       |
| ------------- | ------------------------- | ------------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| **Числовой**  | `/<префикс><N×ранг>`      | `/sh1`, `/sh22`, `/sh333` | Сортировка по частоте; `/sh1` всегда указывает на самый используемый навык; ассоциативное предсказание обновляется каждую сессию |
| **Командный** | `/<префикс>-<имя-навыка>` | `/sh-git-smart`           | Вызов навыков напрямую по имени; порядок меню фиксируется Claude Code по алфавиту, независимо от частоты                         |

Перейдите в **Настройки → Основные → Префикс ярлыка**, введите новое значение и сохраните — ярлыки перестроятся немедленно. Интерфейс обнаруживает конфликты в режиме реального времени и показывает под полем ввода варианты без конфликтов для выбора и применения одним кликом.

**Правила формата:** только буквы, цифры, `-`, `_`; максимум 5 символов.

**Встроенные навыки skill-habit**

Следующие навыки управления встроены как плагин и всегда доступны независимо от префикса:

| Навык                  | Описание                                    | Когда использовать                                                                                                                             |
| ---------------------- | ------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| `/skill-habit:quick`   | Показать текущий префикс и числовые ярлыки  | Забыли префикс или какой номер соответствует навыку? Пример вывода: `Префикс: sh`, `/sh1 → git-smart`                                          |
| `/skill-habit:server`  | Открыть веб-интерфейс управления в браузере | Изменить настройки, просмотреть аналитику, управлять навыками; браузер открывается автоматически, сервер завершается через 5 минут бездействия |
| `/skill-habit:rebuild` | Немедленно пересобрать ярлыки               | После ручного редактирования `~/.skill-habit/config.json`; изменения через Web UI запускают пересборку автоматически                           |
| `/skill-habit:version` | Показать установленную версию               | Уточнить версию при устранении неполадок или сообщении об ошибке                                                                               |

**config.json**

`~/.skill-habit/config.json` — редактируется в интерфейсе или напрямую:

```jsonc
{
  "prefix": "sh",                      // префикс ярлыка: /sh-*
  "top_n": 10,                         // макс. именованных ярлыков за сессию
  "numeric_n": 5,                      // макс. числовых ярлыков (sh1…shN)
  "time_window": "all",                // "today" | "week" | "month" | "all"
  "shortcut_mode": "both",             // "numeric" (/sh1) | "command" (/sh-<name>) | "both"
  "enable_sequence_prediction": true,  // ассоциативное предсказание
  "prediction_n": 5,                   // макс. продвигаемых кандидатов (1–5)
  "top_skills_n": 10,                  // строк в графике топ-навыков
  "pinned_skills": [],                 // всегда первые, независимо от частоты
  "log_retention_days": 30,            // скользящее окно; 0 = хранить бессрочно
  "theme": "system",                   // "light" | "dark" | "system"
  "language": "auto",                  // "zh" | "en" | "de" | "fr" | "ru" | "ko" | "ja" | "auto"
  "analytics_cards": {
    "heatmap": true,                   // сетка активности
    "top_skills": true,                // столбчатая диаграмма частоты
    "transition_graph": true,          // паттерны использования цепочек
    "hourly_distribution": true,       // активность по часам суток
    "weekday_distribution": true       // активность по дням недели
  }
}
```

**🔮 Ассоциативное предсказание (умное упорядочивание)**

Когда `enable_sequence_prediction` включён, при каждом старте сессии система:

1. Читает последний использованный навык
2. Запрашивает историческую матрицу переходов, чтобы предсказать 3 наиболее вероятных следующих навыка
3. Поднимает эти 3 навыка на вершину списка ярлыков

Эффективность зависит от режима ярлыков:

| Режим                        | Эффект                                                                                                                                             |
| ---------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Числовой** (`/sh1 /sh2…`)  | Предсказание напрямую определяет, какой навык занимает `sh1`/`sh2`. Ввод `/sh1` всегда запускает рекомендованный навык. **Полностью эффективно.** |
| **Командный** (`/sh-<name>`) | Порядок меню фиксирован по алфавиту в Claude Code — предсказание **не влияет на порядок в меню**.                                                  |

> Точность улучшается по мере накопления данных. Результаты становятся значимыми после 20+ сессий.

**📐 Порядок числовых ярлыков**

Автодополнение Claude Code ранжирует предложения главным образом по **общей длине имени** — более короткие имена получают более высокий балл и отображаются первыми. skill-habit генерирует числовые ярлыки со **строго возрастающей длиной имён**, повторяя цифру ранга N раз:

| Ранг                   | Ярлык     | Длина      |
| ---------------------- | --------- | ---------- |
| 1 (самый используемый) | `sh1`     | 2 символа  |
| 2                      | `sh22`    | 3 символа  |
| 3                      | `sh333`   | 4 символа  |
| 4                      | `sh4444`  | 5 символов |
| 5                      | `sh55555` | 6 символов |

Это гарантирует, что выпадающий список показывает ярлыки в порядке частоты. Когда паттерны использования меняются, назначения ярлыков автоматически обновляются при следующем запуске сессии.

Для вызова: введите `/sh1` напрямую или `/sh2` (нечёткое совпадение с `sh22`) и нажмите Enter.

## 🖥 Платформа управления

### 🗂 Управление навыками

<a href="../assets/screenshot-skills.png"><img src="../assets/screenshot-skills.png" width="800" alt="Skills tab"></a>

Откройте вкладку **Навыки**, чтобы:

- **Искать и сортировать** — фильтр по имени/описанию; сортировка по числу использований, дате последнего использования или имени (повторный клик меняет порядок на обратный)
- **Редактировать описания** — дважды кликните по описанию для редактирования (или нажмите иконку ✏️ при наведении); Enter или клик в другом месте — сохранение, Esc — отмена
- **Закреплять навыки** — закрепите навык, чтобы он всегда был первым в списке ярлыков; используйте ↑↓ для изменения порядка закреплённых
- **Добавлять в чёрный список** — нажмите **Block** в любой строке, чтобы исключить навык из ранжирования и ярлыков; управление чёрным списком (просмотр, пагинация, разблокировка) — в сворачиваемом разделе «Чёрный список» внизу вкладки
- **История использования** — смотрите, сколько раз использовали каждый навык и когда последний раз

> Во вкладке «Навыки» отображаются только навыки, которые вы использовали хотя бы один раз. Навыки берутся из `~/.claude/skills/` и всех установленных плагинов.

### 📊 Аналитика

<a href="../assets/screenshot-analytics.png"><img src="../assets/screenshot-analytics.png" width="800" alt="Analytics tab"></a>

Вкладка **Аналитика** включает:

- **Тепловая карта активности** — сетка в стиле GitHub; каждая ячейка = вызовы навыков за этот день; диапазон времени настраивается
- **Топ навыков** — столбчатая диаграмма за выбранный период (сегодня / неделя / месяц / всё время)
- **Почасовое распределение** — в какое время суток вы чаще всего используете навыки
- **Распределение по дням недели** — в какие дни вы работаете интенсивнее всего
- **Граф переходов** — какие навыки вы используете совместно (основа для модели предсказаний)

Каждую карточку можно отдельно включить или выключить в Настройках.

**Основные**

| Параметр                         | Описание                                                                                                                         |

| Префикс ярлыков                  | Пространство имён для всех ярлыков, по умолчанию `sh`; обнаруживает конфликты в реальном времени и предлагает свободные варианты |

| Строк в графике топ-навыков      | Количество навыков в столбчатой диаграмме аналитики                                                                              |
| Интервал перестройки             | Минут без активности до автоматической перестройки ярлыков                                                                       |
| Предсказание последовательностей | Предсказывает следующий вероятный навык по истории и поднимает его вверх                                                         |
| Глубина предсказания             | Сколько навыков может продвинуть предсказание (1–5)                                                                              |
| Тема                             | Светлая / Тёмная / Системная                                                                                                     |
| Язык                             | Язык интерфейса: Авто / 中文 / English / Deutsch / Français / Русский / 한국어 / 日本語                                          |
| Переключатели карточек аналитики | Включить/отключить по отдельности: тепловую карту, топ-навыки, граф переходов, почасовое и недельное распределения               |

**Управление данными**

| Параметр              | Описание                                                                              |
| --------------------- | ------------------------------------------------------------------------------------- |
| Дней хранения истории | Скользящее окно хранения (0 = хранить бессрочно); очистка по дням или полное удаление |

**Версия и обновление**

| Статус                       | Поведение                                                                                |
| ---------------------------- | ---------------------------------------------------------------------------------------- |
| Нет обновлений               | Тихая проверка, без уведомлений                                                          |
| Патч-обновление              | Янтарное уведомление только в этой карточке — баннер не отображается                     |
| Мажорное/минорное обновление | Красный баннер вверху страницы с заметками к версии; обновление в один клик или закрытие |
