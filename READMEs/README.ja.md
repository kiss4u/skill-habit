## 問題点

スキルをたくさんインストールした。でも `/` を入力するたびに、リスト全体をスクロールしてから使わなければならない。

## 解決策

skill-habit はあなたが呼び出すすべてのスキルを追跡します（メタデータのみ — プロンプトの内容は一切記録しません）。

セッションが始まるたびに、よく使うスキルを先頭に並べた `/sh-*` ショートカットプレフィックスを再構築します。ラベルはお好みの言語で表示されます。

最もよく使うスキルが `/sh-<your-skill>` に、何週間も使っていないスキルはリストの下に沈みます。

リストは意味のないデフォルト順ではなく、あなたの実際の使用状況を反映します。

## 🔒 プライバシー

skill-habit はプロンプトの内容、ファイルパス、プロジェクト名を**一切記録しません**。

ログエントリの例 (`~/.skill-habit/skill-usage.log`):

```jsonc
{
  "v": 1,                    // スキーマバージョン
  "ts": 1719360000,          // Unixタイムスタンプ
  "skill": "git-smart",      // 名前のみ — その後に入力した内容は除く
  "platform": "claude_code", // スキルを呼び出したAIツール
  "hour": 14,                // 時刻 (0–23)
  "weekday": 1,              // 0 = 月曜日
  "session_id": "abc123",    // 連続するコーディングセッションを識別
  "session_skill_seq": 3,    // セッション内での位置 (1, 2, 3…)
  "project_hash": "a3f2c1",  // 一方向ハッシュ、復元不可
  "args_len": 42             // 文字数のみ、内容は記録しない
}
```

`log_retention_days`（デフォルト: 30）より古いログエントリは、セッションごとに自動削除されます。**設定 → データ管理** から日数を入力して削除（0 = 全削除）することもできます。

## 🚀 クイックスタート

> **必要環境:** macOS または Linux · Python 3.9+ · git
>
> **Windows:** ネイティブはまだ未対応です。回避策として [WSL](https://learn.microsoft.com/windows/wsl/) をご利用ください。

### インストール

**方法A — Claude Code プラグインインストール（推奨）**

```bash
claude plugins install github:kiss4u/skill-habit
```

インストール後 Claude Code を再起動すると、すぐにトラッキングが開始されます。

**方法B — ワンライナーインストール**

```bash
curl -sSL https://raw.githubusercontent.com/kiss4u/skill-habit/main/scripts/bootstrap.sh | bash
```

**方法C — pipx / pip**

> 注意：この方法では組み込みスキル（`/skill-habit:quick`、`/skill-habit:server` など）は自動インストールされません。方法AまたはBを推奨します。

pipx（推奨 — 環境を分離して管理）:
```bash
pipx install git+https://github.com/kiss4u/skill-habit
skill-habit install
```

pip:
```bash
pip install git+https://github.com/kiss4u/skill-habit
skill-habit install
```

> 新しいClaude Codeセッションを開始すると、すぐにトラッキングが始まります。

**その他のプラットフォーム**

Cursor、Codex CLI、Gemini CLIなどへの対応を計画中です。アダプターの追加については「コントリビューション」セクションをご覧ください。

**管理UIを開く**

インストール後、以下のいずれかの方法でダッシュボードを開きます:

Claude Codeから:
```bash
/skill-habit:server
```

pip / pipx インストールの場合:
```bash
skill-habit server
```

bootstrap / cloneインストールの場合:
```bash
python3 ~/.local/share/skill-habit/ui/server.py
```

ブラウザが自動的に開きます。サーバーは5分間操作がなければ自動終了します。

### 設定

**ショートカットプレフィックス**

**ショートカットプレフィックス**は、skill-habit が生成するすべてのショートカットの名前空間です。デフォルトは `sh` です。各セッションで、以下のいずれかまたは両方のモードでショートカットが生成されます（設定で変更可能）:

| モード       | フォーマット             | 例（プレフィックス `sh`） | 備考                                                                                               |
| ------------ | ------------------------ | ------------------------- | -------------------------------------------------------------------------------------------------- |
| **数値**     | `/<prefix><N×rank>`      | `/sh1`、`/sh22`、`/sh333` | 使用頻度順；`/sh1` は常に現在のトップスキルに割り当てられ、関連予測が毎セッション更新              |
| **コマンド** | `/<prefix>-<skill-name>` | `/sh-git-smart`           | スキルを名前で直接呼び出す；Claude Code がアルファベット順に並べるため、頻度順位は位置に影響しない |

**設定 → 一般 → ショートカットプレフィックス** に移動し、新しい値を入力して保存するとショートカットが即時再構築されます。UIはリアルタイムで競合を検出し、入力欄の下に競合のない代替案を表示してクリックで適用できます。

**フォーマットルール:** 使用できる文字は英数字・`-`・`_` のみ。最大5文字。

**組み込みスキル**

以下の管理スキルはプラグインとして組み込まれており、プレフィックスに関わらず常に利用できます：

| スキル                 | 説明                                                     | 使用タイミング                                                                            |
| ---------------------- | -------------------------------------------------------- | ----------------------------------------------------------------------------------------- |
| `/skill-habit:quick`   | 現在のプレフィックスと数字ショートカットマッピングを表示 | プレフィックスや数字の対応を忘れたとき。出力例：`プレフィックス: sh`、`/sh1 → git-smart`  |
| `/skill-habit:server`  | ブラウザでWeb管理UIを開く                                | 設定変更・分析確認・スキル管理に使用。ブラウザが自動で開き、5分操作なしでサーバー自動終了 |
| `/skill-habit:rebuild` | ショートカットをすぐに再生成                             | `~/.skill-habit/config.json` を手動編集した後に使用。Web UIで保存すると自動で再生成される |
| `/skill-habit:version` | インストール済みバージョンを表示                         | 問題調査やバグ報告時にバージョンを確認                                                    |

**config.json**

`~/.skill-habit/config.json` — UIまたは直接編集可能:

```jsonc
{
  "prefix": "sh",                      // ショートカットプレフィックス: /sh-*
  "top_n": 10,                         // セッションごとに生成する名前ショートカットの最大数
  "numeric_n": 5,                      // 数値ショートカットの最大数 (sh1…shN)
  "time_window": "all",                // "today" | "week" | "month" | "all"
  "shortcut_mode": "both",             // "numeric" (/sh1) | "command" (/sh-<name>) | "both"
  "enable_sequence_prediction": true,  // 関連予測
  "prediction_n": 5,                   // 最大昇格候補数 (1–5)
  "top_skills_n": 10,                  // トップスキルグラフの行数
  "pinned_skills": [],                 // 頻度に関わらず常に先頭
  "log_retention_days": 30,            // ローリング保持ウィンドウ; 0 = 永久保持
  "theme": "system",                   // "light" | "dark" | "system"
  "language": "auto",                  // "zh" | "en" | "de" | "fr" | "ru" | "ko" | "ja" | "auto"
  "analytics_cards": {
    "heatmap": true,                   // アクティビティグリッド
    "top_skills": true,                // 頻度棒グラフ
    "transition_graph": true,          // スキル連鎖パターン
    "hourly_distribution": true,       // 時間帯別活動
    "weekday_distribution": true       // 曜日別活動
  }
}
```

**🔮 関連予測（スマート順序付け）**

`enable_sequence_prediction` がオンの場合、セッション開始時に以下を実行します:

1. 最後に使用したスキルを読み取る
2. 過去の遷移マトリクスを参照し、次に使いそうな上位3スキルを予測
3. その3スキルをショートカットリストの先頭に押し上げる

有効性はショートカットモードによって異なります:

| モード                      | 効果                                                                                                                            |
| --------------------------- | ------------------------------------------------------------------------------------------------------------------------------- |
| **数値** (`/sh1 /sh2…`)     | 予測によって `sh1`/`sh2` に割り当てられるスキルが変わります。`/sh1` を入力すると常に推奨スキルが実行されます。**完全に有効。** |
| **コマンド** (`/sh-<name>`) | Claude Codeではメニュー順序がアルファベット順に固定されるため、予測は**メニュー順序に影響しません**。                           |

> データが蓄積されるほど精度が向上します。20セッション以上で意味のある結果が得られます。

**📐 数値ショートカットの順序**

Claude Codeの自動補完は主に**名前の合計長**で候補をランク付けします——名前が短いほどスコアが高く、先に表示されます。skill-habitは各ランクの数字をN回繰り返すことで、**厳密に増加する名前の長さ**を持つ数値ショートカットを生成します：

| ランク                  | ショートカット | 長さ  |
| ----------------------- | -------------- | ----- |
| 1（最も使用頻度が高い） | `sh1`          | 2文字 |
| 2                       | `sh22`         | 3文字 |
| 3                       | `sh333`        | 4文字 |
| 4                       | `sh4444`       | 5文字 |
| 5                       | `sh55555`      | 6文字 |

これにより、ドロップダウンが頻度順にショートカットを表示することが保証されます。使用パターンが変化すると、次のセッション開始時に割り当てが自動的に更新されます。

呼び出し方：`/sh1`を直接入力するか、`/sh2`（`sh22`にファジーマッチ）を入力してEnterを押します。

## 🖥 管理プラットフォーム

### 🗂 スキル管理

<a href="../assets/screenshot-skills.png"><img src="../assets/screenshot-skills.png" width="800" alt="Skills tab"></a>

**スキル** タブでできること:

- **検索 & 並び替え** — 名前・説明でフィルタリング。使用回数・最終使用日・名前で並び替え（同じボタンを再クリックすると逆順）
- **説明の編集** — 説明をダブルクリックしてインライン編集（ホバー時に表示される ✏️ アイコンからも編集可能）。Enterまたは他の場所をクリックして保存、Escでキャンセル
- **スキルのピン留め** — スキルをピン留めするとショートカットの先頭に固定。↑↓でピン留めスキルの順序を変更
- **スキルのブロック** — 任意の行の **ブロック** をクリックすると、そのスキルを頻度ランキングとショートカットから除外。タブ下部の折りたたみ可能なブラックリストセクションで管理（閲覧・ページング・ブロック解除）
- **使用履歴** — 各スキルの使用回数と最終使用日を確認

> スキルタブには、少なくとも1回使用したスキルのみ表示されます。スキルは `~/.claude/skills/` とインストール済みプラグインから取得されます。

### 📊 アナリティクス

<a href="../assets/screenshot-analytics.png"><img src="../assets/screenshot-analytics.png" width="800" alt="Analytics tab"></a>

**アナリティクス** タブには以下が含まれます:

- **アクティビティヒートマップ** — GitHubスタイルグリッド。各セルはその日のスキル呼び出し数。期間は切り替え可能
- **トップスキル** — 選択した期間（今日 / 今週 / 今月 / 全期間）の棒グラフ
- **時間帯分布** — 1日のうちスキルをよく使う時間帯
- **曜日分布** — 最もよくコーディングする曜日
- **遷移グラフ** — よく連続して使うスキルの組み合わせ（予測モデルに活用）

すべてのカードは設定から個別にオン/オフ切り替えが可能です。

### 🛠 設定パネル

<a href="../assets/screenshot-settings.png"><img src="../assets/screenshot-settings.png" width="800" alt="Settings tab"></a>

**設定** タブには3つのカードがあります。変更は即時反映されます:

**一般**

| 設定                         | 説明                                                                                                        |
| ---------------------------- | ----------------------------------------------------------------------------------------------------------- |
| ショートカットプレフィックス | すべてのショートカットスキルの名前空間、デフォルト `sh`；競合をリアルタイムで検出し、競合のない代替案を提案 |
| 時間ウィンドウ               | 頻度ランキングの期間: すべて / 直近6か月 / 今月 / 今週 / 今日                                               |
| ショートカットモード         | 有効にするショートカットの種類: 数値型（`/sh1`）、名前型（`/sh-name`）または両方                            |
| 数値ショートカット数         | 生成する数値ショートカットの最大数（1–9）                                                                   |
| 名前ショートカット数         | 生成する名前ショートカットの最大数                                                                          |
| トップスキルグラフ行数       | Analyticsのトップスキル棒グラフに表示するスキル数                                                           |
| 再構築間隔                   | ショートカットが自動再構築されるまでの非活動時間（分）                                                      |
| シーケンス予測               | 使用履歴から次に使うスキルを予測してリスト上位に移動                                                        |
| 予測深度                     | シーケンス予測が昇格できるスキルの数（1–5）                                                                 |
| テーマ                       | ライト / ダーク / システム                                                                                  |
| 言語                         | UI言語: 自動 / 中文 / English / Deutsch / Français / Русский / 한국어 / 日本語                              |
| 分析カードトグル             | ヒートマップ、トップスキル、遷移グラフ、時間帯・曜日別分布を個別に表示/非表示                               |

**データ管理**

| 設定         | 説明                                                                                   |
| ------------ | -------------------------------------------------------------------------------------- |
| ログ保持日数 | 履歴のローリング保持ウィンドウ（0 = 永久保持）；日数指定で古いデータを削除または全削除 |

**バージョン & アップグレード**

| 状態                  | 動作                                                                     |
| --------------------- | ------------------------------------------------------------------------ |
| 更新なし              | サイレントチェック、通知なし                                             |
| パッチ更新            | このカードのみにアンバー通知 — バナーなし                                |
| メジャー/マイナー更新 | ページ上部に変更履歴の赤いバナー；ワンクリックアップグレードまたは閉じる |
